# Browser Storage State

Saved Playwright browser context states for session reuse.

## Purpose

Automatically generated by `global-setup.ts`:
- Saves authenticated session state (cookies, localStorage, sessionStorage)
- Reused by tests to avoid re-login on each test run
- Speeds up test execution

## Files

- **`storageState.json`** - Auto-generated authenticated browser state (do not commit)

## How It Works

1. **Global Setup** (`globals/global-setup.ts`):
   - Runs once before all tests
   - Performs login and saves session state

2. **Playwright Config** (`playwright.config.ts`):
   - Loads `storageState.json` for each test context
   - Tests start in authenticated state

3. **Re-authentication**:
   - Delete `storageState.json` to force re-login
   - Useful when session expires or you need a fresh login

## Gitignore

Always exclude from version control:

```
test-data/storage-state/storageState.json
test-data/storage-state/*.json
```

## Debugging

Check if storage state exists:

```bash
ls test-data/storage-state/storageState.json
```

Force regeneration:

```bash
rm test-data/storage-state/storageState.json
npm test  # Re-runs global-setup to create fresh state
```
